📋 Development Workflow Scenarios

Scenario 1: Building a Complete Feature

Goal: Build a "Favorites" feature where users can save products

Step 1: Understand Requirements
Prompt: "I need to build a favorites feature where users can save products. User can view saved products and remove favorites. Share the technical requirements and architecture."

Claude analyzes requirements and suggests architecture for:

  • Database schema changes
  • API endpoints needed
  • Frontend components
  • Testing strategy
Step 2: Backend Implementation
Prompt: "Create Laravel API endpoints for favorites: - POST /api/favorites (add favorite) - DELETE /api/favorites/{id} (remove favorite) - GET /api/favorites (list user favorites) Include authorization, validation, and eager loading."
Step 3: Frontend Implementation
Prompt: "Create React component for favorites: - Heart icon button (filled/unfilled states) - Show loading state - Handle optimistic updates - Sync with API - Show success/error messages"
Step 4: Testing
Prompt: "Write tests for favorites feature: - Unit tests for API endpoints - Component tests for UI - E2E tests with Playwright - Test API error scenarios"
Step 5: Commit
Use: /commit to auto-generate commit message

Scenario 2: Debugging a Bug

Issue: Login form shows loading spinner but never submits

Step 1: Gather Information
Share: - Browser console errors - Network tab (API request details) - LoginForm component code - API endpoint code - Environment setup
Step 2: Debug
Prompt: "The login form is stuck on loading. Here's the component code, error message, and network request. What's wrong and how to fix it?"
Step 3: Implement Fix

Claude provides:

  • Root cause analysis
  • Code fix with explanation
  • Prevention strategies
Step 4: Test Fix
Prompt: "Write tests to prevent this regression"
Step 5: Commit
Use: /commit with message like "fix: resolve login form hanging on submission"

Scenario 3: Code Review & Refactoring

Goal: Review PR and suggest improvements

Step 1: Review PR
Use: /review-pr 456 to analyze the PR
Step 2: Check for Issues
Claude checks for: - Security vulnerabilities - Performance problems - Code style violations - Missing tests - SOLID principle violations
Step 3: Request Refactoring
Prompt: "This component is too large. How would you refactor it? Show the refactored version with tests."
Step 4: Apply Changes

Implement refactoring suggestions and commit

Scenario 4: Performance Optimization

Issue: Dashboard page loads slowly

Step 1: Profile Performance
Prompt: "My dashboard is slow. Here's the Lighthouse report and React DevTools profile. What's the bottleneck?"
Step 2: Identify Issues

Claude identifies:

  • Unnecessary re-renders
  • N+1 database queries
  • Large bundle size
  • Missing lazy loading
Step 3: Implement Optimizations
Prompt: "Fix these performance issues: [specific issues]. Include before/after metrics."
Step 4: Verify Improvement
Prompt: "Run Lighthouse again and show improvement in metrics"

💡 Optimization Workflows

Performance Optimization Checklist

Frontend:
☐ Code splitting with React.lazy()
☐ Optimize images (WebP, srcset, lazy loading)
☐ Memoization (useMemo, useCallback, React.memo)
☐ Virtual scrolling for long lists
☐ Bundle analysis and tree shaking
☐ Compression and minification

Backend:
☐ Database indexes on frequently queried columns
☐ Eager loading to avoid N+1 queries
☐ Caching with Redis
☐ Query optimization
☐ Pagination for large datasets
☐ Connection pooling

General:
☐ CDN for static assets
☐ HTTP/2 server push
☐ Gzip compression
☐ Browser caching headers
☐ DNS prefetch
