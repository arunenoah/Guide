Part 3: Claude Code Policy Controls Summary

A. Access Control & Authentication

  • Authorization: Manager approval + Security verification required
  • Role-Based: Junior (restricted), Mid (full), Senior (full), Tech Lead (deployment)
  • Training: 50-question quiz, 85% pass required
  • Revocation: Deactivate + audit + document + log

B. Security & Secrets Management

  • Prohibited: API keys, passwords, tokens, OAuth secrets, DB credentials
  • Controls: .env files, .gitignore, GitHub secret scanning, pre-commit hooks
  • Vault: AWS Secrets Manager or HashiCorp Vault required for production
  • Incident: Report within 1 hour, rotate key, log in security system

C. Data Protection & Privacy

  • PII: Never share customer names, emails, phone numbers
  • Payment: No payment card data, transaction details
  • Production: Never share production data dumps
  • Anonymization: Use anonymized test data for agent analysis

D. Code Quality & Review

  • Segregation: Author ≠ Reviewer (minimum 1, 2+ for critical)
  • Testing: Unit tests 80%+ coverage; staging deployment before production
  • Security: OWASP Top 10 review; no hardcoded secrets
  • Documentation: Meaningful commit messages, git audit trail

E. Agent & MCP Security

  • Permissions: Minimal required; read-only for staging/test data
  • Production: Agents cannot write/modify production
  • Logging: All actions logged (who, what, when, data accessed)
  • Review: Quarterly minimum, continuous monitoring

F. Monitoring & Logging

  • Claude Usage: All sessions logged with timestamps
  • Git Changes: Commit messages form audit trail
  • Incidents: Documented and investigated within 24 hours
  • Retention: 1-year active logs, 3+ year archival

G. Enforcement & Consequences

  • Violations: Escalating consequences (warning → suspension → revocation)
  • Tracking: All violations logged in security system
  • Transparency: Self-reporting results in lighter consequences
  • Appeals: Defined process through management

Part 4: Compliance Audit Procedures

Quarterly Compliance Audit (Required for ISO 27001 & SOC 2)

  • Git Repository Audit: Secret scanning, .env check, credential detection
  • Claude Conversation Sample: 5-10 random conversations per developer
  • Code Review Audit: Last 20 PRs verified for approval + testing
  • Incident Log: All violations reviewed for pattern/response
  • Training Compliance: Verify all developers passed 50-question quiz
  • Agent Activity: Review logs for unauthorized actions
  • Access Control: Verify offboarding, new hires processed correctly
