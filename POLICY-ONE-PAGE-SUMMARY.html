🔐 Claude Code Policy - Quick Reference

Keep this at your desk. Print and reference while using Claude Code.

🛑 NEVER DO THIS

  • Share API keys/secrets
  • Share passwords or tokens
  • Share production data
  • Share customer PII
  • Commit .env with secrets
  • Skip code review
  • Deploy without testing
  • Merge your own code

✅ ALWAYS DO THIS

  • Remove secrets before asking Claude
  • Use .env.example with placeholders
  • Test locally first (lint + tests)
  • Request peer review
  • Deploy staging before prod
  • Understand code before committing
  • Report violations immediately
  • Ask for help when unsure

📋 The Workflow

1.Use Claude (locally)
2.Test & understand code
3.Create feature branch
4.Create PR + request review
5.Wait for approval
6.CI pipeline runs tests
7.Senior merges (not you)
8.Deploy to staging first
9.Tech Lead → Production

⚠️ Red Flags in Code

  • Missing error handling
  • N+1 database queries
  • Plaintext passwords
  • No input validation
  • Missing auth/authorization
  • Hardcoded API keys
  • No exception handling
  • Security assumptions

📢 Violations

Violation 1st 2nd 3rd+
Shared secret Warning Suspend Revoke
Shared data Warning Suspend Revoke
Skipped review Warning Training Escalate
No testing Reject Limited Suspend
