Non-Negotiable Rules (7 Rules for Everyone)

1. No secrets in prompts - Remove API keys, passwords, tokens before asking Claude
2. No copy-paste from .env - Use .env.example with placeholders instead
3. Verify before committing - Run tests locally, understand code, don't blindly commit
4. Get code reviewed - Never merge your own code. Always get 2nd pair of eyes
5. Test before deploying - Deploy to staging first, verify it works there
6. Report violations - If you see secrets leaked, tell security team immediately
7. Stay focused - One feature per session. Don't build 5 things at once

QA-Tester: Playwright & Four Test Types

Why Playwright MCP?

QA-Tester uses Playwright for end-to-end testing that PHPUnit cannot cover: real browser interactions, UI validation, FLKitOver iframe testing, and visual evidence (screenshots/videos). All bugs reported include reproduction steps and visual proof.

Four Required Test Types (Each Feature)

QA-Tester must write all four test types for every feature before approving for release:

1. Positive Tests (Happy Path)

User does everything right. Test the expected flow end-to-end.

// Password reset happy path (Playwright E2E) test('can reset password successfully', async ({ page }) => { await page.goto('/login'); await page.click('text=Forgot Password'); await page.fill('input[type="email"]', 'user@test.com'); await page.click('button[type="submit"]'); // Verify success message await expect(page.locator('text=Check your email')).toBeVisible(); // In PHPUnit: verify database // - Reset token created // - Email sent to SES // - User password unchanged });

2. Negative Tests (Error Cases)

Invalid input, edge cases, constraints violated. Must handle gracefully with clear error messages.

// Test invalid email test('shows error for invalid email', async ({ page }) => { await page.goto('/login'); await page.click('text=Forgot Password'); await page.fill('input[type="email"]', 'invalid-email'); await page.click('button[type="submit"]'); // Verify validation error await expect(page.locator('text=Invalid email')).toBeVisible(); }); // Test rate limiting test('rate limiting prevents brute force', async ({ page }) => { for (let i = 0; i < 11; i++) { const response = await page.context().request.post('/api/password-reset', { data: { email: 'test@example.com' } }); if (i === 10) { expect(response.status()).toBe(429); // Too Many Requests } } }); // Test expired token test('shows error for expired token', async ({ page }) => { // Try to use token older than 1 hour await page.goto('/reset-password?token=expired-token'); // Verify proper error message (not generic 404) await expect(page.locator('text=Token expired')).toBeVisible(); });

3. Non-Functional Tests

Performance, security, accessibility, responsiveness, load testing.

// Performance: API responds under 500ms test('password reset API responds under 500ms', async ({ page }) => { const startTime = Date.now(); const response = await page.context().request.post('/api/password-reset', { data: { email: 'test@example.com' } }); const duration = Date.now() - startTime; expect(duration).toBeLessThan(500); expect(response.ok()).toBeTruthy(); }); // Security: no sensitive data in HTML test('does not expose reset token in HTML', async ({ page }) => { await page.goto('/reset-password?token=secret-token'); const html = await page.content(); expect(html).not.toContain('secret-token'); }); // Accessibility: keyboard navigation works test('keyboard navigation works', async ({ page }) => { await page.goto('/login'); await page.click('text=Forgot Password'); // Tab through form fields await page.keyboard.press('Tab'); await page.keyboard.type('user@test.com'); await page.keyboard.press('Tab'); await page.keyboard.press('Enter'); // Form should submit await expect(page.locator('text=Check your email')).toBeVisible(); }); // Responsive: mobile layout works test('layout works on mobile 375px', async ({ page }) => { await page.setViewportSize({ width: 375, height: 667 }); await page.goto('/login'); await page.click('text=Forgot Password'); // Button should still be clickable, form readable const submitButton = page.locator('button[type="submit"]'); await expect(submitButton).toBeVisible(); await expect(submitButton).toBeInViewport(); });

4. Regression Tests

Ensure existing features still work after changes. Run before every release.
