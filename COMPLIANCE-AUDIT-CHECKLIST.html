1. Code Repository Audit

Scan git history and active code for secrets and sensitive data

1.1 Secret Scanning

Git History Scan for Credentials

Run: git log -p --all | grep -E 'password|api_key|secret|token' (or use truffleHog)

Monthly CRITICAL
GitHub Secret Scanning Enabled

Verify repository has secret scanning enabled in Settings → Security & analysis

Quarterly CRITICAL
.env Files Not Committed

Verify .env files are in .gitignore and not in git history. Only .env.example should be committed.

Monthly CRITICAL
No AWS Keys/Credentials

Scan for: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, STRIPE_SECRET, etc.

Monthly CRITICAL
No Database Passwords

Verify no plaintext database passwords in code/config files

Monthly CRITICAL
No OAuth/API Tokens

Scan for: client_secret, oauth_token, github_token, slack_token

Monthly CRITICAL

2. Claude Conversation Audit

Review Claude Code conversation history for policy violations

2.1 Conversation Content Review (Sample)

Random Sample of 5-10 Conversations Per Developer

Review conversation transcripts for: secrets shared, production data shared, inappropriate requests

Quarterly CRITICAL
No Secrets in Conversations

Check conversations don't contain: API keys, DB passwords, OAuth tokens, private keys

Quarterly CRITICAL
No Production Data Shared

Verify no customer data, emails, payment info, or production database dumps shared

Quarterly CRITICAL
No Proprietary Code Shared

Verify company proprietary algorithms/code not shared (unless approved)

Quarterly HIGH
Conversation Purpose Is Work-Related

All conversations should be for company development, not personal projects

Quarterly MEDIUM

3. Code Review Audit

Verify code reviews are being conducted properly

3.1 PR Review Process

All PRs Have At Least 1 Reviewer Approval

Check last 20 merged PRs - all should have approval before merge

Monthly HIGH
Critical Code Has 2+ Reviews

Security, auth, payment code should have 2 approvals minimum

Monthly HIGH
AI-Generated Code Flagged In PRs

PRs with significant AI code should mention it and have extra scrutiny

Monthly HIGH
All CI/CD Checks Pass Before Merge

Verify: tests pass, linting passes, security scans pass, no coverage regression

Monthly HIGH
No Secrets In PR Code/Diffs

Review PR diffs for accidentally committed secrets

Monthly CRITICAL
Reviewer Comments Show Deep Review

Random sample: Do reviewer comments show they actually read the code?

Quarterly HIGH

4. Security Code Audit

Check for common security vulnerabilities

4.1 OWASP Top 10 Quick Check

SQL Injection Vulnerabilities

Grep for: string concatenation in queries, not using parameterized queries

Quarterly CRITICAL
Authentication Issues

Verify: passwords hashed, JWT validated, session timeouts, rate limiting

Quarterly CRITICAL
Authorization Check

Verify: all protected endpoints check user permissions, not just authentication

Quarterly CRITICAL
Input Validation

Check that user input is validated (length, format, type) on server-side

Quarterly HIGH
XSS Prevention

Verify output encoding, CSP headers, sanitization of user input

Quarterly HIGH
Sensitive Data Exposure

Check: encryption in transit (HTTPS), encryption at rest, no secrets in logs

Quarterly CRITICAL
Dependency Vulnerabilities

Run: npm audit, composer audit, check for known CVEs in dependencies

Monthly HIGH

5. Deployment Audit

Verify proper deployment procedures are followed

5.1 Deployment Process

All Deployments Go To Staging First

Check deployment logs: verify code was tested in staging before production

Monthly HIGH
Only Authorized Personnel Deploy To Production

Verify: only Tech Lead, DevOps team can merge/deploy (not all developers)

Monthly HIGH
Deployment Requires Approval

Check: merge to main requires approval, CI passes, documented deploy logs

Monthly HIGH
Rollback Plan In Place

Verify: team has documented rollback procedure and tests it regularly

Quarterly MEDIUM
Post-Deployment Monitoring

Verify: metrics/logs monitored for 30 mins after deploy for errors

Quarterly MEDIUM

6. Training & Awareness Audit

6.1 Developer Training

All Developers Completed Policy Training

Quarterly HIGH
Junior Developers Have Additional Training

Verify: juniors completed extra security/code review training

Quarterly MEDIUM
Team Understands Policy

Quarterly HIGH
Security Team Training Completed

Verify: security team trained on audit procedures and escalation

Annually MEDIUM

Quarterly HIGH
